#### 自身の役割と成果

一人のメンバーとして、EKSでホスティングされたマイクロサービスをECSに移行する作業を担当しておりました。
また、クラスタの全面はALBでロードバランシングしておりましたが、ELBをNLBに移行したうえでCloudMapによるサービス検出をロードバランサーの背後に置くことで名前解決をする構成に移行していました。

具体的な作業としては、Terraformで定義されていたでEKS x ALBに関する記述を、全てECS x NLB x CloudMapに移行する作業を行なっておりました。

※ あくまで社内からのアクセスに限定しているため、後述の構成でも必要十分な応答速度を満たしつつ、ELBを用いた構成よりもコストが減少できるという観点から、このような構成になっておりました。

#### 当時を振り返って

限られた時間で自身の価値を最大化するため、モブプラミングでは他のジュニアメンバーと交代でコードを書く役割に徹しており、（多少の非難はありましたが）かなり充実した日々だったと記憶しています。

当時はAWSに関する基礎知識が足りていない状態で、「Terraformのコードを編集してください！」という状態であり、それなりのプレッシャーの中で秒単位での成長を実感する毎日でした。

プロジェクトを離れて、AWSのアーキテクト資格を取得する程度の知識がついた今でも、時折当時を回顧して、学ぶべきことはないかと当時のコードをふりかえっています。

#### 現在の近況

前のプロジェクトとは違い、モノリシックな構成のアプリケーションになっていますが、その分ソフトウェアアーキテクチャの設計に目を向けることができました。

参画したての頃を振り返って、ものの数日でプロジェクトの全貌が素人目でわかったのが非常に印象的でした（自慢ではなく、それくらいクリーンであるということを意味してます）。

（機能開発は当然の仕事として、）最近の試みとして、①要望をいかに要件として昇華させるかの工夫、②先人たちが残したプラットフォームのキャッチアップ、③自身の設計を内部ドキュメントとして公開することによる断続的なレビュー、という3つを積極的に行なっております。

そして、その3つの背後にある前提として、UMLやドメイン駆動設計といったベースとなるソフトウェア工学を積極的に意識し始めている最中です。
